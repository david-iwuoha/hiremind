HireMind API Documentation

Simple endpoints for verification, resume parsing and listing verified records.

POST /verify

Verify a transaction anchored credential. JSON body: { "txId": "0.0.XXX@..." }

curl -X POST "https://yourdomain.com/verify" \
  -H "Content-Type: application/json" \
  -d '{"txId":"0.0.6871751@1758623630.688436122"}'

POST /parse-resume

Upload a resume (multipart/form-data). Returns extracted skills and suggested role.

curl -X POST "https://yourdomain.com/parse-resume" \
  -F "resume=@./resume.pdf"

GET /verified

List verified credential records for admin. Returns array of { txId, fileName, sha256, issuer, verifiedAt }.

curl "https://yourdomain.com/verified"

Notes

  • All credential hashes are anchored using Hedera Consensus Service (HCS).
  • Do not store PII on-chain — only store SHA256 / CID proofs.
  • Use Hedera Token Service (HTS) to mint non-transferable credential badges (future).
